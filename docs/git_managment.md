## Процесс обновления версии на гитхабе выглядит так:
1. Проверить статус директории
2. Свериться с предыдущей версией и посмотреть, какие изменения были внесены
3. Посмотреть, какая последняя версия тэга используется в репозитории
4. Добавить информацию о новом пуше в файл versions.md в формате (пример):
   v2.31 (2025-11-04 21:45): desktop idle chrome and swap logic tuning 
   Тэг версии к новому пушу присваивается по следующему принципу: 
   старый тэг + 1, например, если было 1.80, то новый будет 1.81 (пример)
5. При необходимости внести правки в файлы api_usage.md, project_passport.md, directory_tree.md согласно изменениям
6. Добавить в новый коммит все папки и файлы кроме тех, что указаны в .gitignore
7. Закоммитить версию согласно внесенным изменениям
8. Запушить новый коммит и привязанный к нему тэг в ветку указанную ветку
9. Проверить статус директории и убедиться в успешном и полном исполнении предписаний
