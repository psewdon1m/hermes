# Как изучать Hermes

Этот файл — точка входа для всех, кто разбирается в проекте. Ниже короткий маршрут по содержимому каталога `docs/`, чтобы понять, зачем нужен каждый документ и в каком порядке его открывать.

## Пошаговый маршрут

1. **`docs/project_passport.md`** — технический паспорт на русском языке. Дает обзор архитектуры, сервисов и инфраструктуры. С него начинается знакомство.
2. **`docs/api_usage.md`** — контракт REST API: все эндпоинты, форматы запросов/ответов, ошибки и rate limit. Читаем после паспорта, чтобы понимать внешний интерфейс.
3. **`docs/directory_tree.md`** — актуальная структура репозитория с пояснениями, где лежат сервисы, скрипты и шаблоны. Помогает ориентироваться в кодовой базе.
4. **`docs/codex.md`** — правила работы с проектом (что можно/нельзя делать, как оформлять ПР/коммиты). Важно перечитать перед любыми изменениями.
5. **`docs/git_managment.md`** — регламент релизов: как формировать версии, добавлять записи в `docs/versions.md`, тегировать и пушить на GitHub.
6. **`docs/versions.md`** — журнал изменений. Содержит описание всех публичных пушей; сверяемся перед новыми релизами.
7. **`docs/start_here.md`** — текущий файл. Возвращайтесь к нему, чтобы напомнить порядок чтения и назначение каждого документа.

## Дополнительные материалы

- **`README.md`** в корне — быстрое резюме по микросервисам, prerequisites и скриптам.
- **`example.env`** — эталон набора переменных окружения. Используйте его в паре с `docs/api_usage.md` и техпаспортом.
